## 调试

在实际写程序时，我们需要不停的寻找bug。所以，我们有一整套找bug的方式

一、这里用到assert（断言），来进行错误的输出和检查：

```python
def foo(s):
    n = int(s)
    assert n != 0, 'n is zero!'
    return 10 / n

def main():
    foo('0')
```

assert [语句1] [语句2]：语句1如果为True，那么程序可以继续执行，如果断言失败，那么assert语句本身会抛出 AssertionError：

为了避免程序运行时，过多assert的困扰。我们可以在启动Python解释器时，用-O参数来关闭assert，关闭后，可以将所有的assert语句当成pass来看待。

二、用到logging，和assert相比，logging不会抛出错误，而且可以输出文件：

```python
import logging

logging.basicConfig(level=logging.INFO)# 如果不加这一行的话，只会有ZeroDivisionError，没有其他任何信息
s = '0'
n = int(s)
logging.info('n = %d' % n)
print(10 / n)
```

可以看到输出：

```python
$ python err.py
INFO:root:n = 0
Traceback (most recent call last):
  File "err.py", line 8, in <module>
    print(10 / n)
ZeroDivisionError: division by zero
```

三、还可以使用pdb和一些支持断点调试的IDE



## 单元测试

单元测试是用来对一个模块、一个函数或者一个类进行正确性检验的测试工作。一般来说，单元测试是一定数量的测试用例的集成模块。

这种以测试为驱动的开发模式最大的好处就是确保一个程序模块的行为符合我们设计的测试用例。在将来修改的时候，可以极大程度地保证该模块行为仍然是正确的。

编写单元测试，我们需要引入Python自带的`unittest`模块，如下例，我们先编写一个mydict.py

```python
class Dict(dict):

    def __init__(self, **kw):
        super().__init__(**kw)

    def __getattr__(self, key):
        try:
            return self[key]
        except KeyError:
            raise AttributeError(r"'Dict' object has no attribute '%s'" % key)

    def __setattr__(self, key, value):
        self[key] = value
```

接着编写测试脚本mydict_test.py

```python
import unittest

from mydict import Dict

class TestDict(unittest.TestCase):

    def test_init(self):
        d = Dict(a=1, b='test')
        self.assertEqual(d.a, 1)
        self.assertEqual(d.b, 'test')
        self.assertTrue(isinstance(d, dict))

    def test_key(self):
        d = Dict()
        d['key'] = 'value'
        self.assertEqual(d.key, 'value')

    def test_attr(self):
        d = Dict()
        d.key = 'value'
        self.assertTrue('key' in d)
        self.assertEqual(d['key'], 'value')

    def test_keyerror(self):
        d = Dict()
        with self.assertRaises(KeyError):
            value = d['empty']

    def test_attrerror(self):
        d = Dict()
        with self.assertRaises(AttributeError):
            value = d.empty
```

对编写好的单元测试脚本，需要在脚本最后加上：

```python
if __name__ == 'main':
	unittest.main()
```

另一种方法实在命令行中通过参数 -m unittest  直接运行单元测试

```python
$ python -m unittest mydict_test# mydict_test 是自己编写的测试脚本
.....
----------------------------------------------------------------------
Ran 5 tests in 0.000s

OK
```

## setUp和tearDown

在单元测试中编写setUp()和tearDown()方法。这两个方法会分别在每调用一个测试方法的前后执行。

作用：setUp()完成测试需要的程序动作，而tearDown()方法则关闭这一动作。

```python
class TestDict(unittest.TestCase):

    def setUp(self):
        print('setUp...')

    def tearDown(self):
        print('tearDown...')
```

## 文档测试

Python内置的‘文档测试’模块（doctest）模块可以直接提取注释中的代码并进行测试。示例如下：

```python
class Dict(dict):
    '''
    Simple dict but also support access as x.y style.

    >>> d1 = Dict()
    >>> d1['x'] = 100
    >>> d1.x
    100
    >>> d1.y = 200
    >>> d1['y']
    200
    >>> d2 = Dict(a=1, b=2, c='3')
    >>> d2.c
    '3'
    >>> d2['empty']
    Traceback (most recent call last):
        ...
    KeyError: 'empty'
    >>> d2.empty
    Traceback (most recent call last):  #Traceback与后面括号之间一定要加空格，不然会报错。
        ...
    AttributeError: 'Dict' object has no attribute 'empty'
    '''
    def __init__(self, **kw):
        super(Dict, self).__init__(**kw)

    def __getattr__(self, key):
        try:
            return self[key]
        except KeyError:
            raise AttributeError(r"'Dict' object has no attribute '%s'" % key)

    def __setattr__(self, key, value):
        self[key] = value

if __name__=='__main__':
    import doctest
    doctest.testmod()# 调用上文注释中的测试代码
```
